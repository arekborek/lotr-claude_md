# Module Index

Each module configures **one area** of a project's `CLAUDE.md`. A module file
contains two things: the **interview questions** to ask, and the **template** it
renders into `CLAUDE.md`. Each module owns exactly one section of the output, so
modules can be installed independently or upgraded one at a time.

| Module | File | What it configures |
|--------|------|--------------------|
| Git branching policy | `modules/git-branching-policy.md` | Branch model, naming, PR & merge rules |

> **Roadmap:** testing strategy, code style & linting, CI/CD, security & secrets,
> dependency management, release & versioning, environments & configuration.
